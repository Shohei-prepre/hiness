事前インストール

S1
リサーチ　―　会社リスト収集
agent1_research.py
所要時間：約6〜8分 出力：raw_companies.csv

求人広告代理店の「会社名＋公式URL」を複数ソースから収集する。

収集ソース：

  • JESRA（日本人材紹介事業協会）会員 322社　※プロフィールページからURL付きで取得
  • ARNA（全国求人情報協会）リンク集
  • PRtimes 人材紹介キーワード
python agent1_research.py

または run_agent1.bat をダブルクリック

列名内容
company_name会社名
official_url公式URL
source取得元（JESRA / ARNA / PRtimes）
S2
フォーム発見　―　問い合わせフォームURL取得
agent2_enrich.py
所要時間：約30〜40分 出力：enriched_companies.csv

公式URLから問い合わせフォームのURLを自動で探し出す。

探索ロジック：

  • トップページの「お問い合わせ」リンクを優先して探す
  • なければ /contact/ /inquiry/ /お問い合わせ/ など9パターンを総当たり
  • <form> + <textarea> または <input> があればフォームと判定
  • 安全チェック：ドメイン一致 / 会社名検出 / 採用・サポートページ除外
python agent2_enrich.py
列名内容
company_name会社名
official_url公式URL
form_urlフォームURL
statusfound / not_found / skip
source取得元
status = found の会社のみSTEP3に進む（目安：100〜150社）
S3
試運転　―　Playwright dry-run
agent3_dryrun.py
所要時間：約20〜30分 出力：safe.csv

Playwrightでフォームを自動入力し、送信ボタン手前で止まる。入力が成功した会社だけをsafe.csvに記録する。

対応フォーム種別：

  • Contact Form 7（WordPress）
  • WPForms
  • MW WP Form
  • 汎用フォーム（上記以外）

除外（送信しない）：

  • toC企業（個人向けサービス）
  • 公的機関・大企業
  • 採用・サポートページ
  • 第三者プラットフォーム（form-mailer.jp等）
python agent3_dryrun.py
S4
本番送信　―　フォーム送信実行
agent4_submit.py
出力：sent_log.csv
実行前に必ず config.py の自社情報・メッセージ本文を設定すること

safe.csv の会社に対して実際にフォームを送信する。

python agent4_submit.py
列名内容
company_name会社名
form_url送信したフォームURL
resultsuccess / failed
timestamp送信日時
error失敗時のエラー内容
設定
config.py　―　自社情報・メッセージの入力
STEP4実行前に必須
config.py を開いて以下の項目を実際の値に書き換えてください。 
SENDER_INFO = {
    "company":   "（自社名）",
    "name":      "（担当者名）",
    "name_kana": "（ふりがな）",
    "email":     "（メールアドレス）",
    "phone":     "（電話番号）",
    "title":     "（役職）",
    "url":       "（自社URL）",
}

MESSAGE_TEMPLATE = """
（営業メッセージ本文をここに記入）
"""
