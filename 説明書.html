S1
リサーチ　―　会社リスト収集
agent1_research.py　／　--industry で業界を切り替え可能
所要時間：約6〜30分（業界による） 出力：raw_companies.csv

--industry オプションで対象業界を自由に指定できる。業界によって内部で使うスクレイパーが自動的に切り替わる。

求人広告代理店（専用ソース・高精度）

  • JESRA（日本人材紹介事業協会）322社
  • ARNA（全国求人情報協会）
  • PRtimes キーワード検索
  • マイナビパートナー代理店
  • Indeed公認パートナー
→ 約270社・URLあり率が高い

その他の業界（汎用ソース）

  • PRtimes キーワード検索（業界名で検索）
  • Google検索「{業界名} 会社 一覧」
→ 取得件数は業界・キーワードによって異なる
# 求人広告代理店（デフォルト・専用ソース使用）
python agent1_research.py

# 業界を指定する場合
python agent1_research.py --industry 不動産仲介
python agent1_research.py --industry 税理士事務所
python agent1_research.py --industry ITコンサル

# pipeline.py からも同様に指定可能
python pipeline.py --steps 1 2 --industry 不動産仲介
汎用モード（求人広告代理店以外）の注意点： PRtimes・Google検索を使うため、取得件数・精度は業界によって変わります。 「{業界名} 会社 一覧」で結果が多く出るキーワードほど精度が上がります。

source取得元（JESRA / ARNA / PRtimes/{業界} / Google/{業界}）
S2
フォーム発見　―　問い合わせフォームURL取得
agent2_enrich.py
所要時間：約30〜40分 出力：enriched_companies.csv

公式URLから問い合わせフォームのURLを自動で探し出す。

  • トップページの「お問い合わせ」リンクを優先して探す
  • なければ /contact/ /inquiry/ /お問い合わせ/ など12パターンを総当たり
  • <form> + <textarea> または <input> があればフォームと判定
  • 安全チェック：ドメイン一致 / 採用・サポートページ除外 / 第三者プラットフォーム除外
python agent2_enrich.py
列名内容
company_name会社名
official_url公式URL
form_urlフォームURL
statusfound / not_found / skip
source取得元
status = found の企業のみ次のSTEPに進む（目安：100〜150社）
2.5
Geminiパーソナライズ　―　各社専用メッセージ生成
agent_personalize.py　／　Gemini 1.5 Flash 使用
所要時間：約10〜20分 Gemini API 必要 出力：personalized.csv

各社の公式サイトを要約し、Gemini APIで「その会社専用の営業本文（中盤）」を生成する。

メッセージ構成（4ブロック）

固定：書き出し挨拶
Gemini生成：各社に合わせた本文
固定：完全成果報酬の価値訴求
固定：CTA＋署名

Geminiへの指示（自動）

・200文字以内
・「〇〇様は〜」書き出し
・相手の強みに言及
・押しつけがましくない締め
GEMINI_API_KEY が必要： GitHub Secrets に GEMINI_API_KEY を登録するか、ローカルの場合は環境変数に設定してください。 
python agent_personalize.py --industry 求人広告代理店
列名内容
company_name会社名
form_urlフォームURL
company_summary公式サイトから抽出したテキスト要約
personalized_bodyGeminiが生成した本文（中盤）
S3
試運転　―　Playwright dry-run
agent3_dryrun.py
所要時間：約20〜30分 出力：dryrun_results.csv

Playwrightでフォームを自動入力し、送信ボタン手前で止まる。実際に送信せず入力の成否を確認する。

対応フォーム種別（優先順位順）：

  • Contact Form 7（WordPress）
  • WPForms
  • MW WP Form
  • 汎用フォーム（上記以外のフォールバック）
python agent3_dryrun.py

S4
本番送信　―　フォーム送信実行
agent4_submit.py
出力：sent_log.csv
実際にフォームを送信します。実行前に GitHub Secrets の自社情報・Gemini APIキーを必ず設定してください。5秒のカウントダウン後に送信開始します。

STEP3で fill_ok = True だった企業に対して実際にフォームを送信する。送信済み企業は自動でスキップされる。

# 完了後にExcel出力もする
python pipeline.py --steps 1 2 --export
GHA
GitHub Actions でチーム全員が使う方法
Gitアカウントがあれば誰でも実行可能・インストール不要
  • GitHubの対象リポジトリを開く → Settings → Secrets → Actions を開く
  • 以下のSecretsを全て登録する：
    SENDER_COMPANY　SENDER_NAME　SENDER_NAME_KANA　SENDER_EMAIL　SENDER_PHONE　SENDER_TITLE　SENDER_URL　GEMINI_API_KEY
  • Actions タブ → 「営業フォーム自動化パイプライン」→ Run workflow
  • 実行するステップと業界名を選択して Run workflow をクリック
  • 完了後、Artifacts から 営業リスト-*.xlsx をダウンロード
STEP3（ドライラン）まで GitHub Actions で実行可能。
STEP4（本番送信）はローカルでの実行を推奨（意図しない送信を防ぐため）。
設定
自社情報の設定方法
GitHub Secrets（推奨）またはローカルの環境変数

GitHub Secrets（チーム共有・推奨）

Settings → Secrets → Actions に各 SENDER_* と GEMINI_API_KEY を登録する。コードへの記載不要。

ローカル実行時

環境変数に設定するか、config.py の SENDER_INFO に直接記入する。
.env ファイルや config.py に書いたままGitにpushしない。 
SENDER_INFO = {
    "company":   os.getenv("SENDER_COMPANY",   "株式会社ハイネス"),
    "name":      os.getenv("SENDER_NAME",      "山田太郎"),
    "name_kana": os.getenv("SENDER_NAME_KANA", "やまだたろう"),
    "email":     os.getenv("SENDER_EMAIL",     "yamada@hiness.co.jp"),
    "phone":     os.getenv("SENDER_PHONE",     "03-XXXX-XXXX"),
    "title":     os.getenv("SENDER_TITLE",     "営業部長"),
    "url":       os.getenv("SENDER_URL",       "https://hiness.co.jp"),
}
